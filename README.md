# Trailer Fuel Log

A side project built to automate the tracking and logging of trailer data, replacing the manual process of texting information every time a trailer is fueled or checked.

## 🚀 The Purpose

Tracking trailer metrics—like hourmeter readings and fuel consumption—is often done through manual text messages. This process is:

- **Tedious:** Re-typing the same info headers every time.
- **Disorganized:** Hard to look back and find historical data in a text thread.
- **Error-Prone:** Easy to make typos while typing on the move.

This application provides a simple, centralized interface to log trailer numbers, hourmeter stats, and gallons pumped, keeping everything organized in a visual card-based list.

## ✨ Features

- **Quick Entry Form:** Easily input trailer numbers, hourmeter readings, and fuel amounts.
- **Card-Based Visualization:** View all logged trailers as individual, easy-to-read cards.
- **Collapsible Log:** Toggle the trailer list visibility to keep the interface clean when focusing on data entry.
- **Side Project Stack:** Built with [Vite](https://vitejs.dev/) and [React](https://reactjs.org/) for a fast, responsive development experience.

## 🛠️ Tech Stack

- **Framework:** React 19
- **Build Tool:** Vite
- **Styling:** Custom CSS (Flexbox for responsive layouts)

## 🏃 Getting Started

1. **Clone the repo:**
   ```bash
   git clone [your-repository-link]
   ```

   ```bash
   npm install
   ```
3. **Start the development server:**
   ```bash

   ```
